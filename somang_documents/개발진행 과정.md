## 3. 설계

### 1. 시스템 아키텍처

```
[T-Bridge] 서비스의 시스템 아키텍처를 설계해줘. 아래 요소를 포함해서 다이어그램으로 그려줘.
- 문서 위치 : /somang_documents/설계/시스템아키텍처.md
- 주요 사용자 (Actors)
- 클라이언트 애플리케이션 (예: Web, Mobile App)
- 백엔드 시스템 (예: API 서버, BaaS)
- 데이터베이스
- 외부 연동 서비스 (필요시)
- 다이어그램은 mermaid의 C4 Model(Context Diagram) 형식으로 작성해줘.
```

### 2. 데이터베이스 개요

```
[T-Bridge] 서비스의 핵심 개념을 바탕으로, 필요한 주요 데이터베이스 테이블과 그 역할을 정의해줘.
- 문서 위치 : /somang_documents/설계/데이터베이스개요.md
- 서비스의 핵심 도메인을 대표하는 테이블 목록을 제시해줘.
- 각 테이블의 주요 컬럼과 데이터 타입을 예측하여 간단한 설명과 함께 포함해줘.
- 내용은 표(markdown) 형식으로 작성해줘.
```

### 3. 데이터베이스 모델링

```
위에서 정의한 데이터베이스 개요를 바탕으로 SQL DDL(Data Definition Language) 스키마를 작성해줘.

- 각 테이블과 컬럼에 영문으로 주석(Comment)을 추가해줘.
```

- 표로 작성
```

```
